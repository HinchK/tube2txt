# AI Assistant Guide for Tube2Txt v2

This file serves as the system rules, context, and orientation guide for any AI (like Gemini, Claude, or ChatGPT) assisting with this repository. Whenever you begin a new session, refer to this document.

## Project Context
**Name:** Tube2Txt v2  
**Purpose:** A CLI tool to convert YouTube videos and playlists into structured web pages with transcripts and screenshots, with AI-assisted markdown analysis powered by Gemini.  
**Tech Stack:** Bash, Python 3.9+, `yt-dlp`, `ffmpeg`, `google-genai` API.

**Background:** This is a rewrite of a legacy script (`Youtube2Webpage`). The main goal is fast performance, clean architecture, zero-bloat dependencies (no massive frontend frameworks like React/Vue, just plain HTML/Vanilla CSS), and seamless AI content extraction.

## Core Files & Architecture
- `tube2txt.sh`: The main Bash entry point. 
  - Handles parsing arguments.
  - Detects if the URL is a single video or a playlist.
  - Downloads the video and `.vtt` subtitles using `yt-dlp`.
  - Calls `tube2txt.py` to handle HTML generation and AI text creation.
  - Uses `ffmpeg` (with `xargs -P` for parallel execution) to quickly extract screenshots for each timestamp generated by the Python script.
- `tube2txt.py`: The Python worker.
  - Parses VTT files.
  - Communicates with Google's Gemini API (`google-genai`) to generate specialized markdown (`outline`, `notes`, `recipe`, or `technical`).
  - Generates the final `index.html`.
- `styles.css`: The Vanilla CSS file copied into every project.
- `OUTPUT/`: An ignored directory where all generated video projects are inherently deposited.

## Strict Rules & Conventions
1. **Output Directory:** ALL downloaded videos, generated HTML, extracted images, and markdown files MUST go into the `OUTPUT/` directory (e.g., `OUTPUT/my-project/`). The `.gitignore` explicitly ignores `OUTPUT/`.
2. **Performance Constraints:** 
   - Operations should be as fast as possible. Python scripts should be extremely optimized.
   - Do NOT serialize image extraction; always utilize parallelization (e.g., `xargs -P`).
3. **Vanilla Web Tech:** The generated webpage `index.html` uses raw HTML/CSS. If introducing Javascript, use plain Vanilla JS. **Do not** introduce complex frameworks like React or Tailwind unless explicitly instructed.
4. **Environment Variables:** The primary API key required is `GEMINI_API_KEY`. If modifications are made requiring third-party tools, handle missing API keys gracefully (using fallbacks or clear error messages).
5. **Robust Error Handling:** Do not allow the script to fail silently. Ensure all `yt-dlp` and `ffmpeg` outputs/errors are suppressed when successful, but clearly printed and halted if something goes wrong.

## Common Agent Commands
If you need to test the script, use the following template (ensure your API key is exported):
```bash
export GEMINI_API_KEY="your-key-here"
./tube2txt.sh test-video "https://www.youtube.com/watch?v=YOUR_VIDEO_ID" --ai --mode notes
```
Check inside `OUTPUT/test-video/` to verify results.
